## 2. 秘密情報の定義

本宣言にいう「秘密情報」とは、以下の二種類の**NDAによって保護されるゲームクライアントデータ内容**に限定されるものとす：
- miHoYo / Cognosphere の秘密保持契約（NDA）条項に直接拘束される**ゲームクライアント**のデータ内容。
- miHoYo が自発的に公開し提供していない**ゲームクライアント**のデータ内容。

> 当チームは、**機密保持手段としてのNDAの法的正当性を尊重しており**、反対するのは「**NDAによって保護されたゲームクライアント内容に対する漏洩行為**」のみであり、NDA制度そのものに反対するものではない。

本宣言にいう「非秘密情報」とは、既に公開された正式ゲームクライアントのゲームバージョンのデータ内容を指す。このうち、本来秘密情報であったが正式ゲームクライアントの内容と一致するものについては、miHoYo が「正式ゲームクライアント向け事前ダウンロードデータ」を配信せし日より、その秘密属性は解除され、非秘密情報の範疇に属するものとす。但し、当該事前ダウンロードデータに含まれるゲームのシナリオ内容については、法に基づき依然として秘密情報に属するものとす。

> ゲームのシナリオ内容は当ソフトウェアの機能稼働に必須ではなく、故に呼び出すことはなく、その必要もなし。

## 3. 漏洩行為がもたらす実質的損害

当チームは深く承知す：「NDAによって保護されたゲームテストクライエントの内容または未公開のリソースを提前に漏洩せしめる」行為は、決して単なる「情報共有」などではなく、原開発会社（miHoYo / Cognosphere）の多次元にわたる**組織的攻撃および実質的損害**に他ならず、その実態は以下の通りであるが、これに限定されぬ：

### 3.1 直接的な経済的損失

- **「実装前の熱冷め」効果**：プレイヤーがキャラクターのスキルやシナリオを事前に知ることで、「飽き」を感じたり「期待外れ」と思ったりして消費計画を放棄し、それが新キャラクターのガチャの収益の直接的な毀損に繋がる。
- **マーケティング転換への打撃**：漏洩行為は公式のプレ宣伝を無効化し、既に投入せしサンクコストが本来の転換効果を生まず、マーケティングリソースの徒労を招く。

### 3.2 ビジネスモデルと宣伝スケジュールの破壊

- 早期の暴露は公式の当初のマーケティング企画を著しく妨害し、綿密に計画された「サプライズ感」の価値を大幅に貶める。
- 「ゲームのバージョンアップデートを通じて注目度を高める」という核心的運営モデルを破壊し、公式にバージョンプランの一時的な変更を強要し、無用の運営コストを増加せしめる。

### 3.3 商業的競争および安全保障上のリスク

- NDAによって保護された未公開の**ゲームクライアント**データを組織的に発掘し公開せし行為は、核心的な企業秘密（キャラクターデザイン、数値体系、シナリオの展開、バージョンプラン等を含む）を潜在的競争相手に暴露するに等しい。
- 競争相手はその情報を用いて事前の布石を打ち、戦略を調整し、甚だしきはターゲットを絞って引き抜きを行うなど、原開発元の市場の独占的優位を弱体化せしめる恐れあり。

### 3.4 コミュニティ環境の毒化

- **信頼感の崩壊**：漏洩内容は度々文脈を無視して切り取られ、プレイヤーの偏った評価や誤解を生み、コミュニティの紛争を引き起こし、健全な環境を著しく悪化せしめる。
- **サプライズ感の消失**：プレイヤーが事前にシナリオ、キャラクター、イベント等の情報を知破ることで、正式ゲームクライアント実装時の新鮮味や探索の楽しみが著しく低下す。
- **公平性の毀損**：ゲームテストクライエントデータが提前に公開されることで、一部のプレイヤーが情報非対称の優位を得ることとなり、ゲーム内の公平な競争環境を破壊す。

### 3.5 創作意欲の減耗

知的財産権を無視せし暴露行為は、上流のゲームデザイナーや開発チームの創作意欲を著しく挫き、良質なゲーム製品の供給を低下せしめ、更にはゲーム産業全体の発展に悪影響を及ぼすこととなる。

## 4. 当ソフトウェアの自主規制と誓約事項

前条に述べた原開発元への損害を避け、かつ健全なプレイヤー環境を維持せしため、当ソフトウェアは**最高水準の自己コンプライアンス要求**を厳守することを誓約す：

### 4.1 NDAに保護されたテスト用ゲームクライアント内容の収録拒否

当ソフトウェアは、NDAによって保護され未だ正式ゲームクライアントにて公開されていない**ゲームクライアント内容**（キャラクターの数値、シナリオのテキスト、美術リソースおよび将来のバージョンプランを含むがこれに限らず）を**決して保有せず、収録せず、展示せぬものとす**。

### 4.2 NDAに保護されたテスト用ゲームクライアントの不法な探査の厳禁

当チームは、**NDAによって保護された**ゲームテストクライエントの**ゲームクライアント**（非正式ゲームクライアントを含む）に対するクラッキング、デコンパイル、アンパックまたは暗号化メカニズムの回避行為に**決して関与せず**；また、かかる行為に従事するいかなる組織または個人とも、これら技術交流やデータ交換を**行わぬものとす**。

### 4.3 違反の誘引等の厳禁

当チームは、既にNDAに署名せし内部テスト参加者に対し、機密漏洩の**教唆、煽動または誘引を決して行わぬものとす**。

## 5. データソースの合法的裏付けと透明性

当ソフトウェアが利用せしすべてのゲームバージョンに関連するデータおよび素材は、そのソースが**公開され、安全であり、NDAのリスクを伴わぬ**ことを保証す：

### 5.1 公式が自発的に公開せし情報

- 公式がソーシャルメディア（例：HoYoLAB 等）、予告番組または公式告知を通じて自発的に発表せし情報。
- miHoYo 公式がすでに外部に展示せし内容（例：「天外衛星通信」新キャラクター予告、新バージョンの予告番組および宣伝用動画等）。

### 5.2 正式ゲームクライアントにて公開せし内容

- 既に正式にサービスインし、いかなるプレイヤーもアクセス可能な状態にあるリアルタイムのゲームクライアント内容。
- 原則として、ゲーム開発チームが正式サービス開始の１～２日前に提供せし**正式ゲームクライアント向け事前ダウンロードデータ**（ゲームテストクライエントのデータではない）の第三者による探査結果を用いんとするが、以下の制限を設ける：
  1. 当該データに含まれるゲームのシナリオ内容については、ネタバレの恐れを防ぐため、当ソフトウェアが取り扱う範疇には属さぬものとす。当ソフトウェアの機能実装においても、かかる情報を呼び出す必要は一切なし。
  2. 『崩壊：スターレイル』の場合においては、新バージョンが正式公開された当日から利用可能な当該バージョンの内容に限って使用す。
- 若しデータ探査の公開提供者がNDAデータに対する不法な探査に関与している疑いがある場合、当チームはここに宣言す：かかる不法なデータには一切**接触せず、利用せぬものとす**。

### 5.3 秘密データ採用を拒否する技術的理由

前述せし法的リスクに加え、当ソフトウェアが非正式ゲームクライアントの使用を拒否するもう一つの要旨は**技術的信頼性**にあり。非正式ゲームクライアント（ゲームテストクライエント等）の探査データは極めて不確実性が高く、公式は正式稼働前に度々変更を加え得る。これら変動の激しいデータは、当ソフトウェアが精確なキャラクターのステータス計算や安定した補助機能を提供するための信頼し得る根拠とは絶対になり得ぬものとす。

### 5.4 サードパーティAPIの接続

- **キャラクターステータス公開照会インターフェース**（Enka.Network等）：これらのAPIは一切NDAデータにアクセスせず、プレイヤーが自発的に公開せしキャラクター展示データのみを提供するものとす。

## 6. プロジェクト開発の防御メカニズムと貢献者規範

オープンソースおよびコミュニティのプロジェクトとして、当チームはプロジェクトの開発プロセスおよびコミュニティの貢献者に対して**厳格な規範および技術的防御メカニズム**を設く：

### 6.1 技術的防御および審査メカニズム

- **提出の制限およびリソース更新のパイプライン**：当ソフトウェアの GitHub リポジトリにおいて受理するのは、ソースコードの変更を伴う Pull Request（プルリクエスト）のみであり、ゲームバージョンのデータが関与する提出は一切受理せぬ；ゲームリソースの更新については、すべて当チーム内部の専用の継続的インテグレーション（Continuous Integration, CI）パイプラインによって実行され、当該パイプラインにはデータフィルタリングメカニズムが備えられており、これにより許可なく秘密データが生成物へ混入することを阻止す。
- **内容の審査および漏洩防止メカニズム**：当チームは定期的に内容の審査を実施し、データソースの全面的な合法性を確保す。もし潜在的欠陥を発見した場合、直ちに「`contact@pizzastudio.org`」へ連絡せよ。当チームは72時間以内に処置を行うべし。

### 6.2 内容提出の禁止令

何人も、Issue、Pull Request、プロジェクトの議論フォーラムおよびすべての関連コミュニティにおいて、以下の内容を直接提出し、間接的に伝播し、リンクを引用し、または形を変えて暗示せしことを**厳に禁ずる**：

- **ゲームテストクライエントデータ**：Betaテストまたはクローズドテストの関連データ。
- **未実装内容**：未だ公開されていないキャラクター情報、未実装のシナリオテキスト。
- **秘密情報**：いかなる場合も本宣言第2条において定義せし秘密情報。

もし貢献者または使用者が本禁止令に違反した場合、これにより生じうる一切の法的および契約違反の責任を自ら負うものとす。

### 6.3 貢献者の許諾および保証

当プロジェクトのリポジトリにおいてゲームバージョンデータが関与するいかなる Pull Request も受理せぬこと、すなわちソースコード変更の提出に限ることの性質上、参加者（Issue、PRの提出、議論の参加、コードの貢献者を含む）は皆、以下の宣言および保証を行わねばならぬ：

- **内容の合法的裏付け**：提出せしすべての内容（コード、議論、提案を含む）は、本宣言第2条において定義せし如何なる秘密情報も含まぬこと。
- **開発の独自性**：提出せしコードは、NDAによって保護された技術、データまたは情報等にいかなる根拠も求めて開発されたものではなきこと。
- **教唆および流布の禁止**：未だかつて如何なる形式（直接間接を問わず）においても、プロジェクトコミュニティ内にて秘密情報を流布し、引用し、暗示せしことはなく、また他者にNDA条項の違反を教唆せしこともなきこと。
- **損害賠償責任**：万一提出せし内容が前述の保証に反し、これにより当プロジェクトまたは当チームが法的手続きに巻き込まれ、名誉または財産上の損害を被る事態となった場合、違反せし貢献者はすべての法的責任を負い、当チームの被りたる直接的および間接的損害の全てを賠償せし全責任を負うものとす。

### 6.4 機能リクエストに対する処分

もし使用者が当ソフトウェアに対し、「未だ正式に発表されずして情報漏洩のみによって知られる」内容に対する事前の対応（先行実装等）を求めた場合、そのリクエストは**ただちにクローズ (Closed) され、対応を拒否されるものとす**。これに例外はなし。

### 6.5 違反事項に対する裁定

度重なりNDAの関連内容を違反して提出せし者に対し、当チームはこれを**永久に遮断（BAN）**し、当プロジェクトへの参加を拒絶せし権利を有す。当チームはまた、訴訟の際に司法機関へ立証するための用に供すべく、証拠を収集せし権利を留保するものとす。
