## 1. 聲明宗旨與立場

### 1.1 背景與必要性

鑑於近期針對遊戲未公開內容之系統性竊取暨惡意洩密事件（如 HomDGCat/妮可少女 訴訟案，Case No. 1:26-cv-00676-SEG）對整體遊戲社群及開發商造成重大之負面影響，敝團隊爰在此鄭重聲明：

敝軟體與任何竊取、傳播、利用未公開遊戲資料之「內鬼／洩密」行為者概無任何關聯，並就此類違反保密協定 (NDA) 暨侵害營業秘密之**行為**，予以**嚴正切割並劃清界線**。

### 1.2 生效與修訂

敝聲明自發布之日起生效。敝團隊得視法令變更、行業慣例或實務需求，隨時修訂敝聲明，並公告於專案儲存庫。使用者與貢獻者負有定期查閱最新版本之義務。凡繼續使用敝軟體或參與敝專案者，即推定為同意並接受當時有效之聲明條款。

## 2. 保密資訊之定義

敝聲明所指之「保密資訊」，僅限於下列兩類**受 NDA 保護之遊戲客戶端資料內容**：
- 受米哈遊（miHoYo / Cognosphere）保密協定（NDA）條款直接拘束之**遊戲客戶端**資料內容；
- 未經米哈遊主動公開提供之**遊戲客戶端**資料內容。

> 敝團隊**尊重 NDA 作為保密工具之法律正當性**，反對者僅為「**對受 NDA 保護之遊戲客戶端內容的洩密行為**」，而非 NDA 制度本身。

敝聲明所稱「非保密資訊」，係指已公開之正式服遊戲版本資料內容；其中，原屬保密資訊但與正式服內容一致者，自米哈遊推送「正式服預下載資料」之日起，即解除其保密屬性，歸入非保密資訊之範疇。惟前項預下載資料中涉及遊戲劇情之內容，依法仍屬保密資訊。

> 遊戲劇情之内容非敝軟體功能運轉所必需，故不予且無須調用。

## 3. 洩密行為所致之實質損害

敝團隊深知：「將受 NDA 保護之測試服遊戲客戶端內容或未公開資源予以提前洩露」之行為，絕非單純之「資訊分享」，實為對原廠開發團隊（miHoYo / Cognosphere Pte. Ltd.）於多個維度上之**系統性攻擊與實質損害**，其狀包括但不限於：

### 3.1 直接經濟損失

- **「未發先冷」效應**：玩家提前獲知角色技能或劇情後，因感受「倦怠」或「不符預期」而放棄消費計畫，直接導致新角色卡池營收之折損。
- **營銷轉化之衝擊**：洩密行為致使官方預熱宣傳失效，已投入之沉沒成本無法產生應有之轉化效益，造成營銷資源之平白耗損。

### 3.2 商業模式與宣發節奏之破壞

- 提前曝光將嚴重干擾官方原定之行銷企劃，致使精心籌劃之「驚喜感」大幅貶值。
- 破壞「藉由遊戲版本更新以提升關注度」之核心營運模式，迫使官方臨時更動版本規劃，徒增營運成本。

### 3.3 商業競爭與安全風險

- 系統性挖掘並公開受 NDA 保護之非公開**遊戲客戶端**數據，等同將核心商業機密（包含角色設計、數值體系、劇情走向、版本規劃）暴露於潛在競爭對手。
- 競爭對手得利用前揭情資提前布局、調整策略，甚或針對性進行挖角，削弱原廠之市場獨占優勢。

### 3.4 社群生態之毒化

- **信賴感瓦解**：洩密內容屢遭斷章取義，致生玩家片面之評價與誤解，引生社群爭端，嚴重敗壞健康之生態環境。
- **驚喜感消退**：玩家提前悉知劇情、角色、活動等資訊，致正式服上線時之新鮮感與探索樂趣大幅低落。
- **公平性受損**：測試服遊戲客戶端數據提前公開，使部分玩家獲致資訊不對稱之優勢，破壞遊戲內之公平競爭環境。

### 3.5 創作熱忱之貶抑

無視智慧財產權之曝料行為，將嚴重挫傷上游遊戲設計師與開發團隊之創作意願，降低優質遊戲產品之供給，乃至對整體遊戲產業發展生負面之影響。

## 4. 敝軟體之自律規範與承諾事項

為免對原廠造成前條所述之損害，並維護健全之玩家生態，敝軟體承諾恪遵**最高標準之自我合規要求**：

### 4.1 拒絕收錄受 NDA 保護之測試用遊戲客戶端內容

敝軟體**絕不具備、不收錄、亦不展示**任何受 NDA 保護且尚未於正式服遊戲客戶端公開之**遊戲客戶端內容**（包括但不限於：角色數值、劇情文本、美術資源及未來之版本規劃）。

### 4.2 嚴禁對受 NDA 保護之測試用遊戲客戶端的非法探勘

敝團隊**絕不參與**任何針對受 NDA 保護之測試服遊戲客戶端（含非正式服客戶端）之破解、反編譯、解包或規避加密機制之行為；亦**不與**任何從事此類行為之組織或個人進行此類技術交流或此類資料交換。

### 4.3 嚴禁違規誘導

敝團隊**絕不教唆、煽惑或誘導**任何已簽署 NDA 之內部測試人員洩漏機密。

## 5. 資料來源合規與透明化

敝軟體所取用之所有與遊戲版本相關之資料與素材，其來源均確保為**公開、安全、不涉及 NDA 風險**：

### 5.1 官方主動公開之資訊

- 官方藉由社群媒體（如米遊社、HoYoLAB 等）、前瞻直播或官方公告主動發布之資訊。
- 米哈遊官方已對外展示之內容（如「天外衛星通訊」新角色預告、新版本前瞻節目與宣傳影片等）。

### 5.2 正式服公開之內容

- 已正式上線且任一玩家均得存取之即時遊戲客戶端內容。
- 原則上採用遊戲開發團隊於正式開服前一至二日所釋出之**正式服預下載資料**（非測試服遊戲客戶端資料）之第三方探勘結果，惟設有下列限制：
  1. 該資料中涉及遊戲劇情之內容，非屬敝軟體處理之範疇，以防生劇透之虞。敝軟體之功能實作亦毋須調用此類資訊。
  2. 於《星穹鐵道》之情形，現僅限使用新版本開服當日起可利用之該版本內容。
- 倘資料探勘之公開提供方涉有對 NDA 資料進行不法探勘之情事，敝團隊特此聲明：對此類不法資料一概**不予接觸、不予利用**。

### 5.3 拒絕採用保密資料之技術考量

除前揭法律風險外，敝軟體拒絕使用非正式服資料之另一要旨為**技術可靠性**：非正式服（如測試服遊戲客戶端）之探勘數據具高度不確定性，官方於正式上線前仍可能頻繁更動。此等變動無常之數據，無從作為敝軟體提供精確角色面板計算或穩定輔助功能之可靠依據。

### 5.4 第三方應用程式介面（API）之介接

- **角色面板公開查詢介面**（如 Enka.Network 等）：此類 API 概不存取 NDA 資料，僅提供玩家主動公開之角色展櫃數據。

## 6. 專案開發防護機制與貢獻者規範

身為開源暨社群專案，敝團隊對專案之開發流程與社群貢獻者設有**嚴格之規範與技術防護機制**：

### 6.1 技術防護與審查機制

- **提交限制與資源更新管線**：敝軟體之 GitHub 儲存庫僅受理涉及程式碼變更之 Pull Request，概不受理任何涉遊戲版本資料之提交；遊戲資源之更新，悉由敝團隊內部專屬持續整合（Continuous Integration, CI）管線執行，該管線設有資料過濾機制，俾阻絕未經授權之保密資料滲入產出物。
- **內容審查與防漏機制**：敝團隊定期實施內容審查，以確保資料來源全面合規。倘察覺潛在缺失，請逕洽「`contact@pizzastudio.org`」，敝團隊將於 72 小時內處置。

### 6.2 內容提交禁令

嚴禁任何人於 Issue、Pull Request、專案討論區及所有相關社群中，直接提交、間接傳播、引用連結或變相暗示下列內容：

- **測試服遊戲客戶端數據**：Beta 測試或封閉測試之相關資料。
- **未實裝內容**：尚未公開之角色情報、未實裝之劇情文本。
- **保密資訊**：任何經敝聲明第 2 條所界定之保密資訊。

倘貢獻者或使用者違反本項禁令，應自負因此所生之一切法律與違約責任。

### 6.3 貢獻者之授權與保證

鑑於敝專案儲存庫不受理任何涉遊戲版本資料之 Pull Request，僅限程式碼變更類提交，凡參與者（含提交 Issue、PR、參與討論、貢獻代碼者）均應作如下之聲明與保證：

- **內容合規性**：所提交之全部內容（含程式碼、討論、建議等）均未包含敝聲明第 2 條所界定之任何保密資訊。
- **開發原創性**：所提交之程式碼均未以任何受 NDA 保護之技術、數據或資料為基礎進行開發。
- **禁止教唆與散布**：未曾以任何形式（不論直接或間接）於專案社群內散布、引用、暗示保密資訊，亦未教唆他人違反 NDA 條款。
- **損害賠償責任**：倘因提交之內容違反前述保證，致敝專案或敝團隊面臨法律訟累、受有名譽或財產上之損害，違約之貢獻者應承擔悉數法律責任，並對敝團隊所受之直接與間接損害負完全賠償之責。

### 6.4 功能請求之處置

如使用者請求敝軟體提前適配「尚未正式發布且僅見於曝料」之內容，該請求將予以**逕行關閉 (Closed) 並拒絕處理**，概無例外。

### 6.5 違規情事之裁處

屢次違規提交 NDA 相關內容者，敝團隊得予以**永久封鎖**，拒絕其參與敝專案。敝團隊並保留蒐證之權利，以備於涉訟時向司法機關舉證之用。
