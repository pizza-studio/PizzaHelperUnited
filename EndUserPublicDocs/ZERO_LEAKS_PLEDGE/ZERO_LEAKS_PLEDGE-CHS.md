敝声明乃披萨工作室所颁布之合规规范，适用于《拿铁小助手 (The Latte Helper)》（下称「敝软件」），并溯及既往适用于已停止开发之前身《披萨小助手 (The Pizza Helper)》5.x 版本。

## 1. 声明宗旨与立场

### 1.1 背景与必要性

鉴于近期针对游戏未公开内容之系统性窃取暨恶意泄密事件（如 HomDGCat/妮可少女 诉讼案，Case No. 1:26-cv-00676-SEG）对整体游戏社群及开发商造成重大之负面影响，敝团队爰在此郑重声明：

敝软件与任何窃取、传播、利用未公开游戏资料之「内鬼／泄密」行为者概无任何关联，并就此类违反保密协定 (NDA) 暨侵害营业秘密之**行为**，予以**严正切割并划清界线**。

### 1.2 生效与修订

敝声明自发布之日起生效。敝团队得视法令变更、行业惯例或实务需求，随时修订敝声明，并公告于专案储存库。使用者与贡献者负有定期查阅最新版本之义务。凡继续使用敝软件或参与敝专案者，即推定为同意并接受当时有效之声明条款。

## 2. 保密资讯之定义

敝声明所指之「保密资讯」，仅限于下列两类**受 NDA 保护之游戏客户端资料内容**：
- 受米哈游（miHoYo / Cognosphere）保密协定（NDA）条款直接拘束之**游戏客户端**资料内容；
- 未经米哈游主动公开提供之**游戏客户端**资料内容。

> 敝团队**尊重 NDA 作为保密工具之法律正当性**，反对者仅为「**对受 NDA 保护之游戏客户端内容的洩密行为**」，而非 NDA 制度本身。

敝声明所称「非保密资讯」，系指已公开之正式服游戏版本资料内容；其中，原属保密资讯但与正式服内容一致者，自米哈游推送「正式服预下载资料」之日起，即解除其保密属性，归入非保密资讯之范畴。惟前项预下载资料中涉及游戏剧情之内容，依法仍属保密资讯。

> 游戏剧情之内容非敝软件功能运转所必需，故不予且无须调用。

## 3. 泄密行为所致之实质损害

敝团队深知：「将受 NDA 保护之测试服游戏客户端内容或未公开资源予以提前泄露」之行为，绝非单纯之「资讯分享」，实为对原厂开发团队（miHoYo / Cognosphere Pte. Ltd.）于多个维度上之**系统性攻击与实质损害**，其状包括但不限于：

### 3.1 直接经济损失

- **「未发先冷」效应**：玩家提前获知角色技能或剧情后，因感受「倦怠」或「不符预期」而放弃消费计划，直接导致新角色卡池营收之折损。
- **营销转化之冲击**：泄密行为致使官方预热宣传失效，已投入之沉没成本无法产生应有之转化效益，造成营销资源之平白耗损。

### 3.2 商业模式与宣发节奏之破坏

- 提前曝光将严重干扰官方原定之行销企划，致使精心筹划之「惊喜感」大幅贬值。
- 破坏「借由游戏版本更新以提升关注度」之核心营运模式，迫使官方临时更动版本规划，徒增营运成本。

### 3.3 商业竞争与安全风险

- 系统性挖掘并公开受 NDA 保护之非公开**游戏客户端**数据，等同将核心商业机密（包含角色设计、数值体系、剧情走向、版本规划）暴露于潜在竞争对手。
- 竞争对手得利用前揭情资提前布局、调整策略，甚或针对性进行挖角，削弱原厂之市场独占优势。

### 3.4 社群生态之毒化

- **信赖感瓦解**：泄密内容屡遭断章取义，致生玩家片面之评价与误解，引生社群争端，严重败坏健康之生态环境。
- **惊喜感消退**：玩家提前悉知剧情、角色、活动等资讯，致正式服上线时之新鲜感与探索乐趣大幅低落。
- **公平性受损**：测试服游戏客户端数据提前公开，使部分玩家获致资讯不对称之优势，破坏游戏内之公平竞争环境。

### 3.5 创作热忱之贬抑

无视智慧财产权之曝料行为，将严重挫伤上游游戏设计师与开发团队之创作意愿，降低优质游戏产品之供给，乃至对整体游戏产业发展生负面之影响。

## 4. 敝软件之自律规范与承诺事项

为免对原厂造成前条所述之损害，并维护健全之玩家生态，敝软件承诺恪遵**最高标准之自我合规要求**：

### 4.1 拒绝收录受 NDA 保护之測試用游戏客户端内容

敝软件**绝不具备、不收录、亦不展示**任何受 NDA 保护且尚未于正式服游戏客户端公开之**游戏客户端内容**（包括但不限于：角色数值、剧情文本、美术资源及未来之版本规划）。

### 4.2 严禁对受 NDA 保护之封测试用游戏客户端的非法探勘

敝团队**绝不参与**任何针对受 NDA 保护之测试服游戏客户端（含非正式服客户端）之破解、反编译、解包或规避加密机制之行为；亦**不与**任何从事此类行为之组织或个人进行此类技术交流或此类资料交换。

### 4.3 严禁违规诱导

敝团队**绝不教唆、煽惑或诱导**任何已签署 NDA 之内部测试人员泄漏机密。

## 5. 资料来源合规与透明化

敝软件所取用之所有与游戏版本相关之资料与素材，其来源均确保为**公开、安全、不涉及 NDA 风险**：

### 5.1 官方主动公开之资讯

- 官方借由社群媒体（如米游社、HoYoLAB 等）、前瞻直播或官方公告主动发布之资讯。
- 米哈游官方已对外展示之内容（如「天外卫星通讯」新角色预告、新版本前瞻节目与宣传影片等）。

### 5.2 正式服公开之内容

- 已正式上线且任一玩家均得存取之即时游戏客户端内容。
- 原则上采用游戏开发团队于正式开服前一至二日所释出之**正式服预下载资料**（非测试服游戏客户端资料）之第三方探勘结果，惟设有下列限制：
  1. 该资料中涉及游戏剧情之内容，非属敝软件处理之范畴，以防生剧透之虞。敝软件之功能实作亦毋须调用此类资讯。
  2. 于《星穹铁道》之情形，现仅限使用新版本开服当日起可利用之该版本内容。
- 倘资料探勘之公开提供方涉有对 NDA 资料进行不法探勘之情事，敝团队特此声明：对此类不法资料一概**不予接触、不予利用**。

### 5.3 拒绝采用保密资料之技术考量

除前揭法律风险外，敝软件拒绝使用非正式服资料之另一要旨为**技术可靠性**：非正式服（如测试服游戏客户端）之探勘数据具高度不确定性，官方于正式上线前仍可能频繁更动。此等变动无常之数据，无从作为敝软件提供精确角色面板计算或稳定辅助功能之可靠依据。

### 5.4 第三方应用程式介面（API）之介接

- **角色面板公开查询介面**（如 Enka.Network 等）：此类 API 概不存取 NDA 资料，仅提供玩家主动公开之角色展柜数据。

## 6. 专案开发防护机制与贡献者规范

身为开源暨社群专案，敝团队对专案之开发流程与社群贡献者设有**严格之规范与技术防护机制**：

### 6.1 技术防护与审查机制

- **提交限制与资源更新管线**：敝软件之 GitHub 储存库仅受理涉及程式码变更之 Pull Request，概不受理任何涉游戏版本资料之提交；游戏资源之更新，悉由敝团队内部专属持续整合（Continuous Integration, CI）管线执行，该管线设有资料过滤机制，俾阻绝未经授权之保密资料渗入产出物。
- **内容审查与防漏机制**：敝团队定期实施内容审查，以确保资料来源全面合规。倘察觉潜在缺失，请迳洽「`contact@pizzastudio.org`」，敝团队将于 72 小时内处置。

### 6.2 内容提交禁令

严禁任何人于 Issue、Pull Request、专案讨论区及所有相关社群中，直接提交、间接传播、引用连结或变相暗示下列内容：

- **测试服游戏客户端数据**：Beta 测试或封闭测试之相关资料。
- **未实装内容**：尚未公开之角色情报、未实装之剧情文本。
- **保密资讯**：任何经敝声明第 2 条所界定之保密资讯。

倘贡献者或使用者违反本项禁令，应自负因此所生之一切法律与违约责任。

### 6.3 贡献者之授权与保证

鉴于敝专案储存库不受理任何涉游戏版本资料之 Pull Request，仅限程式码变更类提交，凡参与者（含提交 Issue、PR、参与讨论、贡献代码者）均应作如下之声明与保证：

- **内容合规性**：所提交之全部内容（含程式码、讨论、建议等）均未包含敝声明第 2 条所界定之任何保密资讯。
- **开发原创性**：所提交之程式码均未以任何受 NDA 保护之技术、数据或资料为基础进行开发。
- **禁止教唆与散布**：未曾以任何形式（不论直接或间接）于专案社群内散布、引用、暗示保密资讯，亦未教唆他人违反 NDA 条款。
- **损害赔偿责任**：倘因提交之内容违反前述保证，致敝专案或敝团队面临法律讼累、受有名誉或财产上之损害，违约之贡献者应承担悉数法律责任，并对敝团队所受之直接与间接损害负完全赔偿之责。

### 6.4 功能请求之处置

如使用者请求敝软件提前适配「尚未正式发布且仅见于曝料」之内容，该请求将予以**迳行关闭 (Closed) 并拒绝处理**，概无例外。

### 6.5 违规情事之裁处

屡次违规提交 NDA 相关内容者，敝团队得予以**永久封锁**，拒绝其参与敝专案。敝团队并保留搜证之权利，以备于涉讼时向司法机关举证之用。
