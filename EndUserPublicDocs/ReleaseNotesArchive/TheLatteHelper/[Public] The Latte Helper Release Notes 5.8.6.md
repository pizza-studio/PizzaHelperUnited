// 《拿铁小助手》v5.8.6 的更新内容简述：

- 从这个版本开始，在新增使用者壁纸时会被提示裁切第三个尺寸「纵向裁切」。如果您的设备足够新、硬件资源相对于当前运行的作业系统版本而言足够充裕的话，裁切后的壁纸的解析度会比以往高出一倍。这项改动不影响本软件随附的壁纸。同时，在以「指定原始文件」的形式新增使用者壁纸时，现可指定以 AVIF 与 PNG 格式编码的原始文件。
- 使用者壁纸裁剪画面在 macOS 系统下（iPad-app-on-mac）现在会启用一套新的 Slider 控件、不再依赖 Apple 的 Slider 实作。这有助于已经升级到测试版系统的使用者们对本软件的正常使用。
- 对桌面小工具（Desktop Widgets）体系做了一些调整，提前新增了对 Extra Large Portrait 小工具尺寸的支持。但这种小工具尺寸要求当前系统版本高于 26。
- 在 App 处于背景闲置（墓碑）状态时，现在会暂停对 EnkaDB 与 GachaMetaDB 的本机数据库的档案系统状态的检测。

注意：用户拿尚未发行过正式版的年度大版本更新的测试版系统来运行的情形不受敝团队所支持。Apple 研发者会员授权合约限制我们在对 App Store 张贴的版本更新日志当中提及与这类操作系统有关的情形细节（特别是与系统 API 有关的年度行为变化）。我们只能说这种情形可能会导致任何形式的资料丢失。

(本软件不再在 App Store 提供 macCatalyst 版本。这可以减轻 App Store 审委会的审核工作量。这也导致该 App 无法继续支持 Intel Mac 机种，因为只有 Apple Silicon Mac 可以直接运行 iPad 应用。上文中讨论到的与 macOS 有关的内容更新均指该 App 的 iPadOS 版本在 macOS 系统下的行为。本软件以 SwiftUI 技术制作，而 SwiftUI 在 Apple Silicon 电脑上往往能得到更良好的效能体验。)

// Major changes introduced in The Latte Helper v5.8.6:

- Starting from this version, when adding a custom wallpaper you will be prompted to crop a third size: "Vertical Crop." If your device is recent enough and has sufficient hardware headroom relative to the current OS version, the cropped wallpaper resolution will be doubled compared to before. This change does not affect the built-in wallpapers shipped with the app. Additionally, when adding a custom wallpaper via "Specify Original File," AVIF and PNG-encoded source files are now supported.
- The wallpaper cropping interface on macOS (iPad-app-on-Mac) now uses a new custom Slider control and no longer relies on Apple's Slider implementation. This helps users who have already upgraded to a beta OS continue using the app normally.
- Some adjustments have been made to the Desktop Widgets system, with early support added for the Extra Large Portrait widget size. This widget size requires a system version higher than 26.
- When the app is in a suspended (background tombstone) state, file system monitoring for the local EnkaDB and GachaMetaDB databases is now paused.

Note: Scenarios where users run a beta version of an annual major system update that has not yet been officially released are not supported by us. The Apple Developer Program License Agreement restricts us from mentioning details of annual OS-level API behavioral changes related to such operating systems in public release notes for App Store. We can only state that such scenarios may lead to any form of data loss.

(This app no longer supplies a macCatalyst build on the App Store. This helps reduce the App Review workload. As a result, the app can no longer support Intel-based Macs, since only Apple Silicon Macs can run the iPad app directly. All macOS-related mentions above refer to the behavior of the iPadOS version of this app running on macOS. This app is built with SwiftUI, which generally delivers better performance on Apple Silicon machines.)

// Основные изменения в «Латте помощник» v5.8.6:

- Начиная с этой версии, при добавлении пользовательских обоев вам будет предложено обрезать третий размер — «Портретная обрезка». Если ваше устройство достаточно новое и обладает достаточным запасом аппаратных ресурсов относительно текущей версии ОС, разрешение обрезанных обоев будет вдвое выше прежнего. Это изменение не затрагивает встроенные обои, поставляемые с приложением. Кроме того, при добавлении пользовательских обоев через «Указать исходный файл» теперь поддерживаются исходные файлы в форматах AVIF и PNG.
- Интерфейс обрезки обоев в macOS (iPad-app-on-Mac) теперь использует новый собственный элемент управления Slider и больше не зависит от реализации Slider от Apple. Это помогает пользователям, уже обновившимся до бета-версии ОС, продолжать нормально использовать приложение.
- В систему десктопных виджетов (Desktop Widgets) внесены корректировки: добавлена предварительная поддержка размера виджета Extra Large Portrait. Этот размер виджета требует версии системы выше 26.
- Когда приложение находится в фоновом приостановленном состоянии (tombstone), мониторинг файловой системы для локальных баз данных EnkaDB и GachaMetaDB теперь приостанавливается.

Внимание: Сценарии, при которых пользователи запускают приложение на бета-версии ежегодного крупного обновления системы, ещё не выпущенной официально, не поддерживаются нашей командой. Лицензионное соглашение Apple Developer Program ограничивает нас в упоминании деталей изменений поведения API таких операционных систем в публичных заметках к обновлениям для App Store. Мы можем лишь предупредить, что такие сценарии могут привести к потере любых данных.

(Приложение больше не предоставляет сборку macCatalyst в App Store. Это помогает снизить нагрузку на команду App Review. В связи с этим приложение больше не поддерживает компьютеры Mac с процессорами Intel, поскольку только Apple Silicon Mac могут напрямую запускать iPad-приложения. Все упоминания macOS выше относятся к поведению версии этого приложения для iPad, работающей на macOS. Это приложение создано с использованием SwiftUI, который, как правило, обеспечивает более высокую производительность на машинах Apple Silicon.)
